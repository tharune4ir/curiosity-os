# Hacksmith’s Workshop: Scrap-to-System

## The Run Mode

### 1. The Prompt (10 min)
Present the **System Constraint**.
- **The Mission:** Build a "Gravity-Powered Delivery Device" that moves a 50-rupee coin across a 2-meter gap without anyone touching it.
- **The Twist:** It must take *exactly* 10 seconds. Not 9, not 11.
- **The Scrap Box:** Every group gets the same set of "junk" (cardboard tubes, plastic caps, thread, safety pins).

### 2. Build & Log (45 min)
Students build while maintaining the **Causal Build Log**.
- **Log Entry Format:** 
    - *Attempt #X*
    - *What we changed:* (e.g., Added friction to the ramp using a rubber band).
    - *What happened:* (e.g., Coin stopped entirely).
    - *The Insight:* (e.g., Friction is too high; need a 'step' design instead).
- **Teacher Move:** Walk around and point at the logs. "Where is the failure logic? Show me the pivot."

### 3. The Demo (10 min)
One try per group. The class watches the clock.
- **Award:** Best "Log of Failed Logic" (The group that learned the most).
- **Award:** The "Golden Scrap" (The most robust system).

### 4. The Decompile (10 min)
Translate the physical build to **Systems Thinking**:
- "A system is a set of parts that work together to achieve a goal. If you change one part (the ramp), it affects the whole (the time)."
- "Feedback loops: The coin falling too fast was the signal. Your adjustment was the response."
- "Fragility: Which part of your machine is the 'Single Point of Failure'?"

## Success Criteria
Students can point to a specific causal link in their machine (e.g., "If this thread breaks, the whole system stops") and have a log of at least 3 failed iterations.
Artifact: The functioning Scrap Machine + The completed Causal Build Log.
