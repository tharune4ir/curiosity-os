## The Invisible Architecture
Every app you use that 'remembers' you is backed by a database. When you create an IRCTC account, your name, email, password, and booking history are stored in rows of a massive table — just like a very organized Excel spreadsheet. A relational database (like PostgreSQL or MySQL) stores data in multiple linked tables. Your user table links to your bookings table, which links to the trains table. SQL is the language for talking to these tables — asking questions like 'Find all bookings by user ID 4821 from the past 30 days'.

## The Consumer Trap
Every time you search for something on Amazon, you think Amazon is 'searching the internet'. It's not. It's querying an internal database of 350 million products using an extremely optimized SQL-like query that runs in under 100 milliseconds. The 'search bar' is a database query box.
