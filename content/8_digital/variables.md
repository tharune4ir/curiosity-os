## The Invisible Architecture
A variable is like a labeled dabba (tiffin box) that stores information. You can write 'player_score' on the box and put the number 150 inside it. Later, when the player scores again, you open the box, take out 150, add 20, and put 170 back. The box (variable) always has a name and a value. The name is permanent; the value changes as the program runs. Every single computer program — from WhatsApp to BGMI — is essentially thousands of labeled boxes being updated millions of times per second.

## The Consumer Trap
When someone fills in their name and phone number in a Zomato form, they think they're just 'filling in a form'. In reality, a variable called 'user_name' is being assigned their input, and another called 'user_phone' is being assigned their number — to be processed, stored, and used to track every order forever.

## The Builder Hack
A builder realizes that naming variables well is a superpower. The difference between 'x = 150' and 'player_current_score = 150' is the difference between code that no one — not even you — can understand in a week, and code that reads like English.

## The Superpower
You start seeing data everywhere — not just in code. Spreadsheet cells, form fields, game statistics — they are all variables. The world is a system of labeled boxes being updated.

## The First Line Of Code
Open the browser console (F12 > Console). Type: let myName = 'Ravi'; then type: myName. Hit Enter. You just created and read a variable. You just spoke to the machine.
