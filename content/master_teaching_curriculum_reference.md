## How to use this file

1. Follow the curriculum in **sequence order** when you want the pure master run.
2. Update the **Status** and **Notes** columns as teaching progresses.
3. Use **What I teach here** as the lesson-definition column.
4. Use **What this addresses for students** as the pain-point / purpose column.
5. Use **Example / classroom lens** to quickly remember how the node shows up in real life or how to bring it alive in teaching.
6. Add future topics under the correct wing / arena / module / zone without disturbing the rest of the architecture.

## Column guide

| Column | Meaning |
|---|---|
| Seq | Global sequence number from the synced universe |
| Code | Curriculum code stored in the universe taxonomy |
| Node | The teachable atom / topic / practice unit |
| Type | topic, embedded_topic, practice_zone, practice_atom, or outcome |
| What I teach here | Brief description of what the node is actually teaching |
| What this addresses for students | The issue, weakness, or developmental need this node is designed to address |
| Example / classroom lens | A concrete way the node can show up in practice, class, or real life |
| Status | Starter teaching state; editable over time |
| Notes | Blank working space for live teaching remarks, class adjustments, or future expansion |

## 2. W2 — Cognition

**Long name:** The Cognition Payload  
Cognition is the inner-engine wing. It teaches how the mind learns, where thinking breaks, how decisions go wrong, how evidence should update beliefs, and how writing, dialogue, and reflection turn vague thought into clearer judgment.
